# Ion

##  프로젝트 개요

기존 인천과학고의 면불 예약 시스템은 **ISHS ZALI**(노트북실 자리 예약)와 **인곽 면불 관리 프로그램**(면불 신청) 두 가지로 분산 운영되었습니다. 두 프로그램 모두 심각한 **보안 취약점**과 **UX 문제**가 존재했으나 개발자와의 소통 단절로 해결이 불가능한 상태였습니다.

**Project Ion**은 이 두 프로그램을 완전히 대체하는, 보안과 편의성을 모두 갖춘 통합 면불 예약 시스템입니다.

## 기존 프로그램의 문제점

### ISHS ZALI

| 분류 | 문제 |
|------|------|
| 보안 | Google Firestore DB에 인증 없이 읽기·쓰기 가능 → 악성 스크립트로 전체 자리 일괄 조작 가능 |
| 사용 | 로그인 시스템 부재 → 학번 오타로 타인 예약 실패 / 일주일치 선예약 남용 |
| 관리 | 28기 졸업·진학으로 개발자와 소통 불가, 유지보수 중단 |

### 인곽 면불 관리 프로그램

| 분류 | 문제 |
|------|------|
| 보안 | 관리자 인증키 외부 유출 → 누구나 관리자 권한 획득 가능, 미패치 상태 |
| 사용 | 모바일·태블릿 접속 불가 / 매번 동일 정보 수동 입력 |
| 운영 | 교사가 학생 면불 내역을 직접 복붙하여 지맨에 등록 → 누락 사고 발생 |

## 주요 기능

### 1. 면불 신청
- 노트북실 자리 예약 + 면불 신청을 **동시에** 처리
- 노트북실 외 장소 면불 신청 가능
- 담당 교사가 장소·시간·사유를 확인 후 **승인 / 거절**
- 학생이 교사의 승인 상태를 **실시간** 확인 가능
- 교사가 학생 명의 면불을 직접 대신 등록 가능

### 2. 면불 관리
- 학생: 자신의 면불 신청 삭제
- 교사: 신청 목록 확인·승인·거절·삭제·출력

### 3. IonID 계정 관리
- 교사: 계정 활성화 / 비활성화 / 사용 정지
- 교사: 권한 조회·수정, 타 교사 계정 교사 등록
- 학년 자동 진급 처리
- 신규 가입 시 기본 상태 설정

### 4. 기타
- 당일 급식 조회


## 보안 설계

### IonID 비밀번호 보안
- **Bcrypt** 해싱 알고리즘 적용 (솔팅 + 키 스트레칭)
- DB가 유출되더라도 원문 비밀번호 복원 불가
- 인증은 동일 알고리즘 재실행으로 해시값 비교

```
"pwd": "$2a$10$.POAeIjbUPaDYiqs29vA8u..."
```

### 데이터베이스 보안
- 모든 DB에 **고강도 접속 암호** 적용
- **IP 화이트리스트** 방화벽으로 승인된 IP만 접속 허용
- 관리자도 불가피한 경우를 제외하고 DB 직접 접속 금지

### 웹 공격 방어
| 공격 유형 | 대응 |
|---|---|
| SQL Injection | 방어 처리 완료 및 검증 |
| XSS | 방어 처리 완료 및 검증 |
| CSRF | 방어 처리 완료 및 검증 |
| 코드 취약점 | **CodeQL Scan**으로 취약 코드 탐지·수정 |
| 매크로·부정 사용 | **Google reCAPTCHA** 도입, 행동 점수 기반 사전 차단 |

|------|------|
